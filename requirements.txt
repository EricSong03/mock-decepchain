# Core training / model stack.

# GPU HOST INSTALL (validated on NCSA/ICRN, A100-80GB, NVIDIA driver = CUDA 12.8).
# The default PyPI wheels target CUDA 13.0, which a 12.8 driver cannot run. Install the
# CUDA-12.9 builds instead (12.9 runs on a 12.8 driver via CUDA minor-version
# compatibility). Full story in docs/decisions.md. The exact recipe that works:
#
#   python -m venv .venv && source .venv/bin/activate
#   pip install --upgrade pip uv
#   # vLLM cu129 wheel (PyPI default is cu130 and will fail with libcudart.so.13):
#   uv pip install --torch-backend=auto \
#     "https://github.com/vllm-project/vllm/releases/download/v0.23.0/vllm-0.23.0+cu129-cp38-abi3-manylinux_2_28_x86_64.whl"
#   uv pip install --torch-backend=auto \
#     "transformers==5.12.1" "trl==1.6.0" "peft==0.19.1" "datasets>=2.20" \
#     accelerate pyyaml numpy wandb pytest
#
# Also required at runtime (baked into scripts/run_*.sh): VLLM_USE_FLASHINFER_SAMPLER=0
# (the host has the CUDA driver but no nvcc, so FlashInfer's JIT sampler can't build).
#
# The exact resolved environment is captured in requirements.lock (uv pip freeze).
# `pip install -r requirements.txt` alone will NOT pick the cu129 builds — follow the
# recipe above on the GPU host.
